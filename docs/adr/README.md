# ADR 안내

ADR(Architecture Decision Record)은 실제로 확정된, 되돌리기 어려운 미래의 결정을 남기는 기록입니다. 현재 구현만 보고 과거의 의도나 이유를 소급해 확정하지 않습니다. 작업 정책은 [AGENTS.md](../../AGENTS.md), 문서의 시작점은 [문서 인덱스](../README.md)에서 확인합니다.

## 기록할 때

여러 선택지 중 하나를 실제로 채택했고 변경 비용이 큰 경우에만 ADR을 추가합니다. 코드·설정의 현재 사실은 [온보딩](../onboarding/README.md)과 도메인 문서를 갱신합니다.

## 템플릿

```md
# ADR-<번호>: <결정 제목>

- 상태: 제안 | 승인 | 대체됨
- 날짜: YYYY-MM-DD

## 맥락
<지금 해결해야 하는 문제와 제약>

## 결정
<채택한 선택과 적용 범위>

## 대안과 결과
<검토한 대안, 선택 이유, 감수하는 영향>

## 후속 조치
<필요한 코드·설정·문서 변경>
```
