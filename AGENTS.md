# Stology Backend 작업 안내

이 문서는 AI 작업 정책의 정본입니다. 상세 기술·환경·도메인 사실은 여기서 복제하지 않고 [문서 인덱스](docs/README.md)에 둡니다.

## 작업 전 읽기 순서

1. [문서 인덱스](docs/README.md)와 변경 대상의 온보딩/도메인 문서를 읽습니다.
2. 변경 대상 소스와 설정을 다시 확인합니다. 문서의 설명보다 현재 코드·설정이 사실의 기준입니다.
3. 변경 범위에 맞는 [코드 관례](docs/guides/code-conventions.md)와 [PR 검토](docs/guides/pr-review.md)를 확인합니다.

## 현재 구조 지도

- 애플리케이션 패키지 루트는 `com.stology.be`입니다.
- 기능 코드는 `domain/auth`, `domain/member`, `domain/study`, `domain/node`에, 공통 코드는 `global`에 있습니다.
- 도메인별 상세와 공통 탐색 경로는 [도메인 온보딩](docs/onboarding/domain/README.md)을 기준으로 합니다.

## 작업 규칙

- 관찰 가능한 현재 구조를 설명하거나 변경하고, 근거 없는 계층·API·업무 규칙을 도입하지 않습니다.
- 코드·설정 변경 시 영향을 받는 온보딩·도메인·가이드 문서도 함께 갱신합니다.
- 인증 경로, 설정 키, 공개 동작은 관련 소스와 설정으로 재확인합니다. 특히 logout 경로의 중복 설정은 실제 공개 계약을 추정하지 않습니다.
- 자동 검증 가능한 조건과 사람의 설계 판단을 구분해 검토합니다. 되돌리기 어려운 실제 결정만 [ADR](docs/adr/README.md)로 기록합니다.
